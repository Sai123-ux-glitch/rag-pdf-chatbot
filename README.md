# 📄 Chat with your PDF — RAG-based Document Q&A

A web application that lets you upload any PDF and ask questions about it in plain English. Answers are generated by a Large Language Model, but grounded in the contents of *your* document using Retrieval-Augmented Generation (RAG) — so responses are accurate and cite the passages they came from, instead of being made up.

🔗 **Live demo:** https://rag-pdf-chatbot-lefccoxgiujzkxgaahwcek.streamlit.app

---

## What it does

Upload a PDF, wait a few seconds for it to be indexed, then ask questions. The app finds the most relevant passages in your document and uses them to answer — and shows you exactly which passages (and page numbers) it used.

This solves a real problem with plain LLMs: they hallucinate, and they don't know the contents of your private documents. RAG fixes both by retrieving real context before answering.

---

## How it works

The app runs in two phases:

**Indexing phase (when you upload a PDF):**

1. **Load** — the PDF is read and its text extracted
2. **Chunk** — the text is split into overlapping passages (~1000 characters each) so each piece is small enough to search and feed to the model
3. **Embed** — each chunk is converted into a vector (a numerical representation of its meaning) using a sentence-transformer model that runs locally
4. **Store** — all the vectors are stored in a FAISS index for fast similarity search

**Query phase (when you ask a question):**

5. **Retrieve** — your question is embedded and compared against all stored vectors; the top 4 most similar chunks are returned
6. **Augment** — those chunks are inserted into a prompt as context, alongside your question
7. **Generate** — the LLM reads the prompt and produces an answer based only on the provided context

## Tech stack

| Component | Technology |
|---|---|
| UI & hosting | Streamlit / Streamlit Community Cloud |
| Orchestration | LangChain |
| Embeddings | sentence-transformers (`all-MiniLM-L6-v2`) |
| Vector store | FAISS |
| LLM | Llama 3.1 (8B) via Groq |
| PDF parsing | pypdf |

## Running it locally

**1. Clone the repository**

```bash
git clone https://github.com/Sai123-ux-glitch/rag-pdf-chatbot.git
cd rag-pdf-chatbot
```

**2. Create and activate a virtual environment**

```bash
python3 -m venv venv
source venv/bin/activate        # On Windows: venv\Scripts\activate
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

**4. Add your Groq API key**

Create a free key at [console.groq.com](https://console.groq.com), then create a `.env` file in the project root:

```
GROQ_API_KEY=your_key_here
```

**5. Run the app**

```bash
streamlit run app.py
```

The app opens at `http://localhost:8501`.

## Notes

- The embedding model (~80MB) downloads on first use, so the very first question may take 30–60 seconds. It's cached afterward.
- The API key is never committed to the repository — it's read from a local `.env` file when running locally, and from Streamlit secrets when deployed.
